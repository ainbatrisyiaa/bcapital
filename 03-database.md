# 03 — Database Schema

This database schema supports:

- RAG document storage (PDF + Website)
- PGVector embeddings with metadata filtering
- Session-based chat memory (no login)
- Feedback logging for evaluation

Database: **PostgreSQL + PGVector**

---

## 1. Required Extensions

```sql
CREATE EXTENSION IF NOT EXISTS vector;
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```

---

## 2. Documents Table

Stores all approved knowledge sources (PDF and Website).

```sql
CREATE TABLE documents (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    source_type TEXT NOT NULL,           -- 'pdf' or 'web'
    title TEXT NOT NULL,
    source_url TEXT,                    -- for web pages (nullable for PDF)
    file_path TEXT,                     -- for PDFs (nullable for web)
    content_hash TEXT,                  -- detect content changes
    created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 3. Chunks Table (Core of RAG)

Each document is split into chunks for embedding and retrieval.

```sql
CREATE TABLE chunks (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    document_id UUID REFERENCES documents(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    page_number INT,                    -- for PDFs
    heading TEXT,                       -- section heading if exists
    metadata JSONB,                     -- {project, state, property_type, access_level}
    embedding VECTOR(1536),             -- OpenAI embedding dimension
    created_at TIMESTAMP DEFAULT NOW()
);
```

### Recommended metadata format

```json
{
  "project": "Austin Heights",
  "state": "Johor",
  "property_type": "Condo",
  "access_level": "public"
}
```

---

## 4. Chat Sessions (Memory Without Login)

```sql
CREATE TABLE chat_sessions (
    id UUID PRIMARY KEY,                -- generated by frontend
    started_at TIMESTAMP DEFAULT NOW(),
    last_active TIMESTAMP DEFAULT NOW()
);
```

---

## 5. Chat Messages (Conversation History)

```sql
CREATE TABLE chat_messages (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    session_id UUID REFERENCES chat_sessions(id) ON DELETE CASCADE,
    role TEXT NOT NULL,                 -- 'user' or 'assistant'
    message TEXT NOT NULL,
    sources JSONB,                      -- citations used in assistant reply
    created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 6. Chat Summaries (Optional Optimization)

Used when sessions become long to reduce token usage.

```sql
CREATE TABLE chat_summaries (
    session_id UUID PRIMARY KEY REFERENCES chat_sessions(id) ON DELETE CASCADE,
    summary TEXT,
    updated_at TIMESTAMP DEFAULT NOW()
);
```

---

## 7. Feedback Table

Tracks user feedback for evaluation and improvement.

```sql
CREATE TABLE feedback (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    message_id UUID REFERENCES chat_messages(id) ON DELETE CASCADE,
    rating INT NOT NULL,                -- 1 = 👍, -1 = 👎
    comment TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

---

## 8. Indexes for Performance

```sql
CREATE INDEX idx_chunks_embedding
ON chunks USING ivfflat (embedding vector_cosine_ops);

CREATE INDEX idx_chunks_metadata
ON chunks USING GIN (metadata);

CREATE INDEX idx_messages_session
ON chat_messages (session_id);
```

---

## 9. Retrieval Logic Using This Schema

When a user asks a question:

1. Convert question into embedding
2. Query `chunks` using vector similarity
3. Optionally filter using metadata (state, project, type, access_level)
4. Retrieve top-k chunks
5. Use `document_id` to construct citations (PDF page or URL)
